В проекте были реализованы модульные тесты для проверки основных частей серверной логики.

Файл auth-utils.test.js проверяет работу вспомогательных функций авторизации:
1. Хеширование пароля и проверку правильного пароля.
2. Обработку пустого пароля.
3. Возврат false при неправильных или отсутствующих аргументах.
4. Создание и проверку JWT-токена.
5. Извлечение Bearer-токена из заголовка Authorization.

Файл error.middleware.test.js проверяет middleware обработки ошибок:
1. Возврат статуса 400 при ошибках валидации.
2. Возврат статуса 409 при ошибке дублирования данных.
3. Возврат серверной ошибки с кодом 500 уровня и запись ошибки в консоль.

Файл auth.controller.test.js проверяет контроллер авторизации:
1. Успешную регистрацию пользователя со статусом 201.
2. Ошибку 401 при отсутствии Bearer-токена.
3. Получение данных текущего пользователя при корректном токене.

Файл level.service.test.js проверяет сервис уровней:
1. Создание 10 стандартных уровней при пустой базе данных.
2. Получение списка опубликованных уровней из кэша.
3. Загрузку уровней из базы данных при пустом кэше.
4. Запрет запуска неопубликованного уровня.
5. Успешный запуск опубликованного уровня и получение runToken.
